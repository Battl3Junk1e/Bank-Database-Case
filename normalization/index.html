Third Normal Form (3NF) Compliance

The Big Bank Database is designed to adhere to Third Normal Form (3NF), which ensures the elimination of data redundancy and promotes data integrity. The database follows the principles of 3NF by satisfying the following criteria:

  1. First Normal Form (1NF):
    All tables have a primary key, and each column contains atomic (indivisible) values. There are no repeating groups or arrays, ensuring that each piece of data is stored in its simplest form.

  2. Second Normal Form (2NF):
    All non-key attributes are fully functionally dependent on the primary key. This is achieved by eliminating partial dependencies. For example, the Account table contains only information directly related to the account, while other customer details (like name, address, etc.) are stored in the Customer table.

  3. Third Normal Form (3NF):
    All non-key attributes are non-transitively dependent on the primary key. In other words, no non-key attribute is dependent on another non-key attribute. For example, in the Account table, the AccountTypeID is used to reference the AccountType table, ensuring that details about the account type are not redundantly stored in the Account table.

By following 3NF, the database eliminates unnecessary duplication of data, ensures data integrity, and maintains flexibility for future extensions or modifications.
